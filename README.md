# Mapa Pogody W Trakcie Rowerowania

Next.js app for checking route weather while cycling.

## Development

Install dependencies and start the dev server:

```bash
npm install
npm run dev
```

Open http://localhost:3000.

Useful checks:

```bash
npm run lint
npm run build
```

## Next.js 16 Notes For Agents

This project uses Next.js 16 with Turbopack. Before changing Next.js config or APIs, read the local docs in:

```text
node_modules/next/dist/docs/
```

Do not rely on older Next.js conventions from memory.

## Troubleshooting: Turbopack, OneDrive, and `.STARTUP`

The project path includes `OneDrive\.STARTUP`. Turbopack's SQLite-backed filesystem cache can fail in paths like this and make `npm run dev` print `Ready`, then fail with errors such as:

```text
Error: ENOENT: no such file or directory, mkdir '...\MapaPogodyWTrakcieRowerowania\C:\Users\...\Temp\next-mapapogody\dev'
ERR_CONNECTION_REFUSED http://localhost:3000/
```

The expected fix is already in `next.config.ts`:

```ts
experimental: {
  turbopackFileSystemCacheForDev: false,
}
```

Keep that setting unless the project is moved out of OneDrive / `.STARTUP` and the dev server has been retested.

Do not fix this by setting `distDir` to an absolute Windows path or to a path outside the project, such as:

```ts
distDir: "C:\\Users\\...\\Temp\\next-mapapogody"
distDir: "../../next-mapapogody"
```

Next.js 16 docs say `distDir` should not leave the project directory, and Windows absolute paths can be treated incorrectly when joined with the project path.

## Safe Cache Cleanup

If dev/build behaves strangely after config changes, it is safe to stop the dev server and delete `.next`:

```powershell
Remove-Item -LiteralPath .next -Recurse -Force
npm run dev
```

`.next` is ignored by Git and is regenerated by Next.js. Because the repo is inside OneDrive, deleting `.next` may trigger a OneDrive notification about thousands of removed files. That is expected for build/cache artifacts and does not mean source files were deleted.

Before assuming source files were removed, check Git:

```bash
git status --short
```

Only tracked source files shown as `D ...` are real Git deletions.
